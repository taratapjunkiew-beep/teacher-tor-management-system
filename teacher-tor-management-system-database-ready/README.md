## ความสามารถหลัก
- ครูกรอกเพียงรหัสวิชาและชื่อวิชา

## หมายเหตุ
PDF บางไฟล์มีตัวอักษรเพี้ยนหรือแบ่งบรรทัดแปลก ระบบจึงมีสถานะ “ควรตรวจเพิ่ม” ให้แอดมินตรวจในเมนูจัดการรายวิชา


## ระบบแก้คำเพี้ยนจาก PDF
เวอร์ชันนี้เพิ่มตัวทำความสะอาดข้อความก่อนนำเข้า เช่น
- กลุ!ม → กลุ่ม
- อาชีพชี → อาชีพ
- ชั่วชั่โมง → ชั่วโมง
- การเรียนรี รู้ → การเรียนรู้
- เทคโนโลยีสารสนเทศ ยี → เทคโนโลยีสารสนเทศ
- เครือขาย → เครือข่าย
- ผูใช → ผู้ใช้

ระบบจะพยายามรักษาความหมายต้นฉบับ ไม่แต่งเนื้อหาหลักสูตรใหม่ และจะแจ้ง “ควรตรวจเพิ่ม” ถ้าข้อมูลสำคัญยังขาดหรือมีคำเพี้ยนหลุดอยู่


## เวอร์ชัน Full Detail Import
เวอร์ชันนี้แก้ปัญหาสำคัญ:
- PDF มีรายวิชาซ้ำ 2 จุด คือหน้ารายชื่อวิชาและหน้าคำอธิบายรายวิชา
- ระบบเดิมอาจจับรหัสจากหน้ารายชื่อก่อน ทำให้ไม่มีจุดประสงค์/สมรรถนะ/คำอธิบาย
- ระบบใหม่นี้จะอ่านทุกตำแหน่งของรหัสวิชา แล้ว merge ตามรหัสวิชา
- ถ้าพบหน้ารายละเอียดเต็ม ระบบจะใช้ชุดข้อมูลที่มีคำอธิบายรายวิชา จุดประสงค์ และสมรรถนะครบกว่า

## อัปเดตล่าสุด: ฟอร์ม Word ตามแม่แบบ IoT
เวอร์ชันนี้แก้ตัวสร้างไฟล์ Word ให้ยึดรูปแบบจากไฟล์ตัวอย่าง “โครงการสอนการออกแบบส่วนติดต่อประสานสำหรับงาน IoT.docx” เป็นหลัก ได้แก่
- หน้าปกมีตราอาชีวะด้านบนกึ่งกลาง
- หน้าปกและหัวข้อจัดตำแหน่งตามแบบ
- ใช้ TH SarabunPSK ทั้งเอกสาร
- ช่วงต้นเอกสารเป็นแนวตั้ง และตารางใหญ่เป็นแนวนอน
- ตารางมาตรฐานอาชีพมีหัวซ้อนและช่องรหัส PC ตามแบบ
- ตารางวิเคราะห์พฤติกรรมแบบที่ 1 มีหัวตารางซ้อนและช่อง “เฉลี่ยคะแนน”
- ตารางวิเคราะห์พฤติกรรมแบบที่ 2 ใช้ AP ตามแบบ
- ตารางหน่วยการเรียนรู้มีหัว “เวลาเรียน (ชม.)” รวม 3 ช่อง

หมายเหตุ: ระบบจะไม่สร้างรหัส PC เพิ่มเติม หากไม่มีในเอกสารต้นฉบับ จะใส่ “-” เพื่อไม่แต่งข้อมูลหลักสูตรเอง
